## 快速上手

### 启动应用

```bash
# 确保已安装依赖并配置 .env 文件
python web_ui.py

# 浏览器访问 http://127.0.0.1:7860
```

### 基础流程（3步走）

```
步骤1：上传简历 → 步骤2：评估简历 → 步骤3：开始面试
```

### 准备工作

**必需**：
- ✅ PDF 格式的简历文件（推荐 2-3 页）

**可选（提升体验）**：
- 目标岗位的完整 JD（Job Description）
- 明确的求职意向和岗位要求

## 功能详解

### 一、简历管理

#### 功能说明
上传并解析 PDF 简历，为后续评估和面试做准备。

#### 操作步骤

**1. 上传简历**
```
点击「📄 简历管理」Tab 
→ 点击「选择简历文件」按钮
→ 选择 PDF 文件
→ 点击「📤 开始加载简历」
→ 等待 2-5 秒
→ 查看加载状态
```

**2. 使用示例简历**（无需上传）
```
点击「📄 使用示例简历」按钮
→ 系统自动加载内置示例
→ 适合快速体验功能
```

#### 加载结果示例
```markdown
## ✅ 简历加载成功！

| 项目 | 信息 |
|------|------|
| 📄 文件名 | 张三_Python工程师.pdf |
| 📏 文件大小 | 128.5 KB |
| 📝 内容长度 | 3,245 字符 |
| ⏱️ 加载耗时 | 1.23 秒 |
```

#### 注意事项
- ⚠️ 仅支持 PDF 格式（不支持 Word、图片）
- ⚠️ 确保 PDF 是文本型（非扫描件）
- ⚠️ 建议文件大小 < 10MB
- 💡 每次上传会覆盖之前的简历

### 二、简历评估

#### 功能说明
AI 从 6 个维度对简历进行专业评估，提供量化评分和改进建议。

#### 评分维度（总分 100 分）

| 维度 | 分值 | 考察内容 |
|------|------|----------|
| 基本信息完整性 | 0-10 | 联系方式、个人信息、求职意向 |
| 工作经验相关性 | 0-10 | 工作经历匹配度、职业发展连贯性 |
| 项目经验质量 | 0-10 | 项目描述详细度、技术难度、个人贡献 |
| 技能匹配度 | 0-10 | 技能栈匹配、技能深度与广度 |
| 教育背景 | 0-10 | 学历层次、专业相关性 |
| 整体印象 | 0-10 | 排版格式、语言表达、专业度 |

**总分计算**：各维度平均分 × 10（满分 100 分）

#### 三种评估模式

**模式 1：完整评估分析** ⭐️ 推荐

```
适用场景：首次优化简历、准备重要面试
评估时间：30-60 秒
输出内容：
  ✓ 6 个维度详细评分与评价
  ✓ 总体评分（0-100 分）
  ✓ 3-5 条主要优点
  ✓ 3-5 条改进建议
```

**操作步骤**：
```
1. 进入「🔍 简历评估」Tab
2. （可选）填写「目标岗位」，例如：高级 Python 工程师
3. （可选）填写「岗位要求」，例如：
   - 3 年以上 Python 开发经验
   - 熟悉 Django/Flask 框架
   - 有大型项目经验
4. 点击「📊 完整评估分析」按钮
5. 查看进度条（正在准备评估 → 分析简历 → 调用 AI 模型 → 生成报告）
6. 查看详细评估报告
```

**模式 2：快速评分** ⚡️

```
适用场景：快速了解简历水平、多次迭代优化
评估时间：10-20 秒
输出内容：
  ✓ 总体评分
  ✓ 简短评价（2-3 句）
```

**操作步骤**：
```
直接点击「⚡ 快速评分」按钮
```

**模式 3：改进建议** 💡

```
适用场景：已有评分，需要具体优化方案
评估时间：20-40 秒
输出内容：
  ✓ 针对性改进建议（5-8 条）
  ✓ 每条建议包含：问题描述 + 优化方案
```

**操作步骤**：
```
点击「💡 改进建议」按钮
```

#### 评估报告示例

```markdown
# 📊 简历评估报告

## 评分详情

| 维度 | 评分 | 评价 |
|------|------|------|
```

### 三、岗位解读

#### 功能说明
基于岗位 JD 和你的简历，AI 自动生成针对性的面试问题。

#### 操作步骤

**1. 准备岗位 JD**
```
前往招聘网站（如拉勾、Boss直聘、智联招聘）
→ 找到目标岗位
→ 复制完整的岗位描述（包括岗位职责、任职要求、技能要求等）
```

**2. 生成面试问题**
```
1. 进入「🎯 岗位解读」Tab
2. 将 JD 内容粘贴到「岗位描述」输入框
3. 调整「生成问题数量」滑块（建议 10-15 个）
4. 点击「🔍 生成面试问题」按钮
5. 等待 30-60 秒
6. 查看岗位分析和问题列表
```

#### 输出内容

```markdown
# 🎯 岗位解读与面试问题

## 岗位核心要求分析

**技术要求**：
- Python 开发 3 年以上
- 熟悉 Django/Flask 框架
- 了解微服务架构

**软技能要求**：
- 良好的团队协作能力
- 较强的问题解决能力

## 基于您简历生成的面试问题（共 10 题）

### 技术类问题

**Q1: 请介绍一下你在项目X中使用的微服务架构？**
- 考察点：微服务设计、技术选型

**Q2: 你如何优化 Django 应用的性能？请结合实际案例说明。**
- 考察点：性能优化经验、问题解决能力

### 项目类问题

**Q3: 项目Y中提到 QPS 提升 3 倍，具体是如何实现的？**
- 考察点：性能优化、量化思维

### 行为类问题

**Q4: 描述一次你在团队中遇到技术分歧的情况，你是如何处理的？**
- 考察点：沟通能力、团队协作

---
⏱️ 分析耗时: 52.3秒 | 🤖 模型: gpt-3.5-turbo
```

#### 使用建议
- 📌 生成的问题可作为面试准备清单
- 📌 建议针对每个问题准备 2-3 分钟的回答
- 📌 可将问题复制到面试模拟环节进行练习

### 四、模拟面试

#### 功能说明
AI 扮演面试官，进行真实的多轮对话面试，帮助你提升临场表现。

#### 面试类型

| 类型 | 图标 | 侧重点 | 适用场景 |
|------|------|--------|----------|
| 技术面试 | 💻 | 技术能力、问题解决、代码实现 | 技术岗位一面、二面 |
| 行为面试 | 🤝 | 软技能、团队协作、沟通能力 | HR 面、主管面 |
| 综合面试 | 🎯 | 技术 + 行为全面评估 | 终面、CEO 面 |

#### 操作步骤

**1. 配置面试**
```
1. 进入「💼 模拟面试」Tab
2. 在左侧选择「面试类型」（技术/行为/综合）
3. 勾选「🌐 启用联网搜索」（可选，用于验证答案）
4. 点击「🎬 开始面试」按钮
```

**2. 查看开场白**
```
AI 面试官会发送开场白，例如：

"你好！我是今天的面试官。首先欢迎你来参加面试。
我看到你有 3 年的 Python 开发经验，主要使用 Django 和 Flask 框架。
今天我们主要聊聊你的技术背景和项目经验。

那么，能否先简单介绍一下你最近的一个项目？"
```

**3. 回答问题**
```
在底部输入框输入你的回答
→ 点击「📤 发送」或按 Enter 键
→ 等待 AI 面试官回复（10-30 秒）
→ 继续下一轮对话
```

**4. 查看面试统计**
```
点击「📊 查看总结」按钮
→ 查看对话轮数、消息数量等统计信息
```

**5. 结束面试**
```
点击「🗑️ 清空对话」按钮
→ 重新开始新一轮面试
```

#### 对话示例

```
👤 候选人：
我最近的项目是一个电商平台的后端系统，使用 Django 框架开发。

3. 数据库做了读写分离和分库分表

这些措施让系统 QPS 从 500 提升到了 2000+。

🤖 面试官：
不错！我注意到你提到了分库分表，能详细说说你们是按什么维度分片的吗？

```

#### 高级功能

## 使用技巧

### 简历准备
1. ✅ 使用标准的简历模板，避免过度设计
2. ✅ 确保 PDF 文本可被选中和复制
3. ✅ 简历内容控制在 2-3 页
4. ✅ 使用清晰的章节标题（工作经历、项目经验、技能清单等）

### 评估优化
1. ✅ 首次使用先做「完整评估」，了解整体情况
2. ✅ 根据评估建议修改简历后，使用「快速评分」验证改进效果
3. ✅ 如果有明确目标岗位，填写岗位信息可获得更精准的评估

### 面试练习
1. ✅ 技术面试建议开启「联网搜索」，提升真实性
2. ✅ 回答时尽量具体，包含量化数据和技术细节
3. ✅ 可以先在「岗位解读」生成问题，再到面试环节练习
4. ✅ 每次面试建议 15-20 分钟，保持专注

### 迭代优化
```
简历上传 → 完整评估 → 修改简历 → 快速评分 → 
生成面试问题 → 模拟面试 → 总结改进 → 再次练习
```

## 常见问题

### Q1: 支持哪些简历格式？
**A**: 目前仅支持 PDF 格式。Word 格式请先转换为 PDF。

### Q2: PDF 简历解析失败怎么办？
**A**: 可能原因：
- PDF 是扫描件（图片型）→ 请使用文本型 PDF
- PDF 有加密保护 → 请解除保护后重试
- PDF 格式损坏 → 尝试重新导出 PDF

### Q3: 评估时间过长怎么办？
**A**: 
- 评估时间取决于 LLM API 响应速度
- 如果超过 2 分钟仍无响应，刷新页面重试
- 检查网络连接和 API Key 是否有效

### Q4: 能否保存评估报告？
**A**: 目前需要手动复制评估结果。后续版本将支持导出 PDF/Word 报告。

### Q5: 面试对话能否保存？
**A**: 系统目前不持久化对话历史。建议重要对话手动截图或复制保存。

### Q6: 联网搜索是否必需？
**A**: 
- 非必需，默认关闭
- 建议在技术面试或需要验证答案准确性时开启
- 会略微增加响应时间

### Q7: 多次评估同一简历，结果会不同吗？
**A**: 
- 可能略有差异（LLM 生成具有随机性）
- 总体评分和核心建议基本一致
- 建议以第一次完整评估为准

### Q8: 能否评估英文简历？
**A**: 可以，但系统提示词为中文，评估结果也为中文。英文支持后续版本完善。

### 3. 岗位解读

#### 功能说明
根据目标岗位的JD和你的简历，智能生成针对性的面试问题清单。

#### 操作步骤

1. **准备岗位JD**
   - 从招聘网站复制完整的岗位描述
   - 包括：岗位职责、任职要求、技能要求等

2. **进入岗位解读页面**
   - 点击「🎯 岗位解读」标签

3. **输入岗位信息**
   - 将复制的JD内容粘贴到「岗位JD」输入框
   - 建议包含完整信息，越详细越好

4. **设置问题数量**
   - 使用滑块选择生成的问题数量（5-20个）
   - 建议选择10-15个

5. **生成问题**
   - 点击「🔍 生成面试问题」按钮
   - 等待分析完成（约30-60秒）

6. **查看结果**
   - 系统会生成包含以下内容的报告：
     - 岗位信息分析
     - 简历匹配度评估
     - 详细的面试问题清单
     - 每个问题的考察点和评分维度

#### 问题类型

生成的问题通常包含：

- **技术基础**（30%）：核心技术栈和基础知识
- **项目经验**（40%）：结合简历中的项目深入提问
- **问题解决**（20%）：考察解决实际问题的能力
- **综合素质**（10%）：学习能力、团队协作等

#### 使用建议

- 生成问题后，建议提前准备答案
- 可以针对每个问题做笔记
- 反复练习重点问题
- 将问题清单用于后续的模拟面试

### 4. 模拟面试

#### 功能说明
与AI面试官进行多轮对话，模拟真实面试场景，提升临场表现。

#### 面试类型

系统提供3种面试模式：

**1. 技术面试**
- 重点考察专业技能和技术深度
- 基于简历中的技术栈和项目提问
- 包括算法、系统设计等问题
- 适合：技术岗位的准备

**2. 行为面试**
- 重点考察软技能和价值观
- 使用STAR方法引导回答
- 关注团队协作、沟通能力等
- 适合：综合素质评估

**3. 综合面试**
- 技术和行为问题穿插
- 全面评估候选人
- 更接近真实面试场景
- 适合：最终面试准备

#### 操作步骤

1. **进入模拟面试页面**
   - 点击「💼 模拟面试」标签

2. **选择面试设置**
   - 选择面试类型（技术/行为/综合）
   - 决定是否启用「联网搜索」
     - 启用：面试官可以搜索最新信息验证你的回答
     - 不启用：纯粹基于简历内容提问

3. **开始面试**
   - 点击「🎬 开始面试」按钮
   - AI面试官会做简短自我介绍并提出第一个问题

4. **回答问题**
   - 在「你的回答」输入框中输入答案
   - 点击「发送」按钮或按Enter键提交
   - AI面试官会根据你的回答进行追问或提出新问题

5. **持续对话**
   - 进行多轮对话，模拟真实面试流程
   - 系统会自动管理对话历史（最多保留20轮）

6. **查看总结**
   - 随时点击「📊 面试总结」查看统计信息
   - 包括面试类型、对话轮数、消息数量等

7. **重新开始**
   - 点击「🗑️ 清空历史」可清除对话记录
   - 或直接点击「🎬 开始面试」重新开始新一轮面试

#### 回答技巧

**STAR方法**（适用于行为面试）：
- **S**ituation（情境）：描述当时的背景情况
- **T**ask（任务）：说明你的任务目标
- **A**ction（行动）：详细说明你采取的行动
- **R**esult（结果）：总结最终结果和收获

**技术问题回答要点**：
- 先给出简洁的核心答案
- 然后展开说明原理和细节
- 结合实际项目经验举例
- 如果不确定，诚实说明并给出思路

#### 使用建议

- **循序渐进**：从简单问题开始，逐步增加难度
- **真实对待**：把AI当作真实面试官，认真回答
- **记录重点**：遇到不会的问题，记下来后续学习
- **反复练习**：同一类型面试可以多次练习
- **时间控制**：单次面试建议控制在20-30分钟
- **总结反思**：面试后总结表现好和需改进的地方

## 常见问题

### Q1: 简历上传后没有反应？
**A**: 请检查：
- 文件格式是否为PDF
- PDF是否可正常打开
- 文件大小是否过大（建议<10MB）
- 尝试重新上传

### Q2: 评估结果不准确怎么办？
**A**: 可能的原因：
- 简历信息不够完整，建议补充细节
- 未填写目标岗位，导致评估缺乏针对性
- 可以尝试填写岗位信息后重新评估

### Q3: 岗位解读生成的问题太多/太少？
**A**: 
- 可以调整问题数量滑块（5-20个）
- 问题数量建议：初级岗位8-10个，高级岗位12-15个
- 可以多次生成，选择合适的问题组合

### Q4: 模拟面试时AI面试官没有回复？
**A**: 请检查：
- 是否已点击「开始面试」按钮
- 网络连接是否正常
- 查看浏览器控制台是否有错误信息
- 尝试刷新页面重新开始

### Q5: 如何提高评估和面试的质量？
**A**: 建议：
- 使用性能更好的LLM模型（如GPT-4）
- 简历内容尽可能详细完整
- 岗位JD信息尽可能完整
- 面试回答时提供具体的例子和数据

### Q6: 可以保存面试记录吗？
**A**: 
- 目前面试记录保存在浏览器内存中
- 可以手动复制重要的对话内容
- 建议面试后做笔记总结

### Q7: 联网搜索功能有什么用？
**A**: 
- 面试官可以搜索最新技术信息
- 验证你提到的技术概念
- 更真实地模拟有经验的面试官
- 建议在熟练后启用，增加挑战性

## 最佳实践

### 📝 简历优化流程

1. **上传简历** → 2. **完整评估** → 3. **记录问题** → 4. **修改简历** → 5. **再次评估** → 6. **确认优化效果**

### 🎯 岗位针对性准备

1. **收集岗位JD** → 2. **岗位解读分析** → 3. **准备问题答案** → 4. **模拟技术面试** → 5. **模拟行为面试** → 6. **总结提升**

针对每个目标岗位都重复这个流程。

### 💼 面试练习计划

**第一阶段（熟悉阶段）**
- 技术面试 × 2次
- 行为面试 × 2次
- 不启用联网搜索

**第二阶段（提升阶段）**
- 综合面试 × 3次
- 启用联网搜索
- 记录不会的问题

**第三阶段（实战阶段）**
- 综合面试 × 5次
- 启用联网搜索
- 严格时间控制
- 总结优化

- **简历优化**：2-3小时（1-2个工作日）
- **岗位准备**：每个岗位1-2小时
- **面试练习**：每天1-2次，持续1-2周
- **最佳练习时间**：面试前1-2周开始
