## 🚀 快速开始

### 第一步：环境准备

1. **安装 Python**
   - 确保安装了 Python 3.9 或更高版本
   - 检查版本：`python3 --version`

2. **获取 API Key**
   - 选择一个 LLM 服务商（推荐 DeepSeek 或 Qwen，性价比高）
   - 注册账号并获取 API Key
   - 常用服务商：
     - OpenAI: https://platform.openai.com/
     - DeepSeek: https://platform.deepseek.com/
     - Qwen: https://dashscope.aliyun.com/

3. **安装依赖**
   ```bash
   cd interview-coach
   pip install -r requirements.txt
   ```

### 第二步：配置系统

1. **创建配置文件**
   ```bash
   cp .env.example .env
   ```

2. **编辑 .env 文件**
   ```ini
   # 必填：你的 API Key
   LLM_API_KEY=sk-your-api-key-here
   
   # 根据你的服务商选择
   # DeepSeek 示例
   LLM_API_BASE=https://api.deepseek.com
   LLM_MODEL=deepseek-chat
   
   # 或 Qwen 示例
   # LLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
   # LLM_MODEL=qwen-turbo
   ```

3. **验证配置**
   - 保存文件后，确保没有语法错误
   - API Key 前后不要有空格

### 第三步：启动系统

**方法 1：使用启动脚本（推荐）**
```bash
chmod +x start.sh
./start.sh
```

**方法 2：直接运行**
```bash
python3 web_ui.py
```

启动成功后，访问 http://localhost:7860

## 📖 功能详解

### 1. 简历管理

#### 上传简历

1. 点击「📄 简历管理」标签页
2. 点击上传区域，选择 PDF 格式的简历
3. 点击「📤 加载简历」按钮
4. 等待系统解析（通常 1-3 秒）

**注意事项**：
- 仅支持 PDF 格式
- 文件大小建议不超过 10MB
- 确保 PDF 是文本格式（非扫描图片）

#### 查看简历内容

- 上传成功后，会在「简历内容预览」区域显示前 500 字符
- 可以查看文件名、大小、加载时间等信息
- 如果内容显示异常，可能是 PDF 格式问题

### 2. 简历评估

#### 完整评估

1. 确保已上传简历
2. （可选）填写目标岗位，如「Python 后端工程师」
3. （可选）填写岗位要求，如「3年以上经验，熟悉Django...」
4. 点击「📊 完整评估」按钮
5. 等待 LLM 生成评估报告（通常 10-30 秒）

**评估报告包含**：
- 各维度评分（0-10分）
- 评分理由
- 总体评分（0-100分）
- 优点总结
- 需要改进的地方
- 针对性建议

#### 快速评分

1. 点击「⚡ 快速评分」按钮
2. 等待 5-10 秒获得总分和简短评价
3. 适合快速了解简历质量

#### 改进建议

1. 点击「💡 改进建议」按钮
2. 获得详细的优化建议
3. 建议涵盖内容、表达、格式等多个方面

**使用技巧**：
- 可以多次评估，对比不同版本的简历
- 填写目标岗位会得到更精准的评估
- 根据建议修改简历后，重新上传评估

### 3. 模拟面试

#### 准备面试

1. 确保已上传简历
2. 选择面试类型：
   - **技术面试**：侧重技术能力考察
   - **行为面试**：侧重软技能和经验
   - **综合面试**：技术+行为全面考察

3. 选择是否启用「联网搜索」
   - 启用：面试官可以搜索验证你的回答
   - 不启用：纯基于 LLM 知识库

#### 开始面试

1. 点击「🎬 开始面试」按钮
2. AI 面试官会给出开场白并提出第一个问题
3. 在「你的回答」输入框中输入回答
4. 点击「发送」或按 Enter 键提交
5. AI 会根据你的回答继续提问或追问

#### 面试技巧

**回答建议**：
- 尽量详细、具体地回答问题
- 对于技术问题，可以说出自己的理解和经验
- 对于行为问题，使用 STAR 法则（情境、任务、行动、结果）
- 不确定的问题，可以坦诚说明并表达学习意愿

**对话管理**：
- 系统会自动保留最近的对话历史
- 点击「🗑️ 清空历史」可以重新开始
- 点击「📊 面试总结」查看面试统计

#### 联网搜索

启用联网搜索后，面试官可能会：
- 搜索你提到的技术或概念
- 验证事实性信息
- 获取最新的行业动态
- 基于搜索结果提出新问题

**优点**：更接近真实面试，考察更全面
**缺点**：响应时间稍长（多 2-5 秒）

### 4. 系统设置

查看当前系统配置：
- LLM 模型
- API 地址
- 联网搜索状态
- 日志级别等

## 💡 使用技巧

### 简历优化流程

1. **初次评估**
   - 上传当前简历
   - 进行完整评估
   - 记录评分和主要问题

2. **针对性修改**
   - 根据改进建议修改简历
   - 重点优化低分维度
   - 注意保持信息真实性

3. **再次评估**
   - 上传修改后的简历
   - 对比前后评分变化
   - 继续优化直到满意

4. **针对岗位优化**
   - 填写目标岗位信息
   - 根据评估调整简历重点
   - 突出相关经验和技能

### 面试准备策略

1. **熟悉简历内容**
   - 确保能详细说明每一项经历
   - 准备好项目细节和数据
   - 回顾技术栈和技能点

2. **分类练习**
   - 先进行技术面试练习
   - 再进行行为面试练习
   - 最后进行综合面试

3. **反复训练**
   - 同一类型面试可以多次练习
   - 每次清空历史重新开始
   - 注意总结高频问题

4. **记录问题**
   - 记下不会回答的问题
   - 查资料后再次练习
   - 准备标准答案模板

### 提升面试表现

1. **技术深度**
   - 不仅说「会」，还要说「怎么用」
   - 提供具体的使用场景和经验
   - 主动说明遇到的问题和解决方案

2. **逻辑清晰**
   - 先说结论，再说过程
   - 分点阐述，条理清晰
   - 适当举例说明

3. **态度积极**
   - 对不熟悉的技术表示学习意愿
   - 强调团队协作和沟通能力
   - 展现问题解决能力

## ❓ 常见问题

### Q1: 上传简历后显示「简历内容为空」？

**可能原因**：
- PDF 是扫描件（图片格式）
- PDF 有密码保护
- PDF 格式不标准

**解决方法**：
- 使用文字版 PDF（可复制文本）
- 移除 PDF 密码
- 转换 PDF 格式或重新生成

### Q2: 评估/面试响应很慢或超时？

**可能原因**：
- API Key 无效或余额不足
- 网络连接问题
- API 服务器响应慢

**解决方法**：
- 检查 API Key 是否正确
- 检查账户余额
- 尝试更换网络或使用代理
- 切换到其他 LLM 服务商

### Q3: 评估结果不准确或不合理？

**可能原因**：
- 简历信息不完整
- LLM 理解偏差
- 模型能力限制

**改善方法**：
- 完善简历内容，提供更多细节
- 多次评估取平均值
- 尝试使用更强大的模型（如 GPT-4）
- 人工结合评估结果判断

### Q4: 面试问题太简单或太难？

**调整方法**：
- 在简历中调整经验年限描述
- 选择不同的面试类型
- 更换 LLM 模型（不同模型风格不同）
- 在回答中表现出相应水平

### Q5: 联网搜索经常失败？

**可能原因**：
- 网络不稳定
- 搜索引擎被墙或限制
- 达到搜索频率限制

**解决方法**：
- 暂时关闭联网搜索
- 配置代理
- 等待一段时间后重试
- 更换搜索引擎（修改 .env 配置）

### Q6: 如何更换 LLM 模型？

**步骤**：
1. 编辑 .env 文件
2. 修改以下配置：
   ```ini
   LLM_API_KEY=新的-api-key
   LLM_API_BASE=新的-api-地址
   LLM_MODEL=新的-模型名称
   ```
3. 保存文件
4. 重启 Web UI

## 🎯 最佳实践

### 简历准备

1. **保持真实**
   - 不要夸大或虚构经历
   - 系统评估基于内容真实性

2. **重点突出**
   - 突出与目标岗位相关的经验
   - 量化成果和贡献

3. **格式规范**
   - 使用清晰的结构
   - 保持一致的格式风格
   - 避免过度设计

### 面试训练

1. **循序渐进**
   - 从简单的问题开始
   - 逐步增加难度
   - 定期回顾和总结

2. **真实模拟**
   - 把每次练习当作真实面试
   - 认真思考后再回答
   - 注意表达的逻辑性

3. **持续改进**
   - 根据反馈调整回答
   - 不断完善答案模板
   - 扩展知识储备

### 隐私保护

1. **敏感信息处理**
   - 测试时可以隐去真实姓名、联系方式
   - 不要在系统中保存敏感数据
   - 定期清理日志文件

2. **数据安全**
   - 妥善保管 API Key
   - 不要在公共网络使用
   - 建议本地部署使用
